Реализовано:

- `FAQ`
  - поиск по ключевому слову;
  - поиск по полям `question` и `answer`;
  - поиск без учета регистра;
  - нестрогий поиск через `Fuse.js`, чтобы переживать мелкие опечатки;
  - поиск по разделам;
  - вывод вопроса и ответа;
  - кнопки `Назад` и `Главное меню`.
- `Обратная связь`
  - пользователь может отправлять текст и файлы;
  - первая заявка за день подтверждается сообщением;
  - пока заявка открыта, новые сообщения прикрепляются к ней же;
  - при создании новой заявки все администраторы получают уведомление.
- `Админский режим`
  - кнопка `Посмотреть заявки` доступна только роли администратора;
  - список открытых заявок выводится в формате `id - текст`;
  - для кнопок списка используется только `ID` заявки;
  - текст превью ограничен первыми `32` символами;
  - администратор может отправить пользователю текст или файл;
  - пересылка Telegram-сообщения не используется, бот отправляет новое сообщение от себя;
  - заявку можно закрыть.
- `Возможности чат-бота`
  - на входе появился отдельный раздел `Возможности`;
  - бот показывает информацию о платформе, компетенциях, практиках, контактах, спринтах и октокоинах.
- `Управление задачами и сроками`
  - добавлен внутренний HTTP API для событий внешней платформы;
  - поддерживаются уведомления о принятии и отклонении задач;
  - поддерживаются напоминания о неактивности;
  - поддерживаются уведомления о завершении компетенций;
  - поддерживаются уведомления о спринтах;
  - поддерживаются уведомления о новых задачах.
- `Рассылки`
  - пользователь может отдельно включать и выключать маркетинговые рассылки;
  - пользователь может отдельно включать и выключать новостные рассылки;
  - есть API для массовой отправки по подписчикам.
- `Инфраструктура`
  - `Node.js`;
  - `Express` с health-check эндпоинтом `/health`;
  - `Sequelize`;
  - `PostgreSQL`;
  - запуск через Docker.

Реализация:

- основное ТЗ требует `Node.js + Express + Sequelize + node-telegram-bot-api`;
- дополнительные требования требуют `PostgreSQL` и упоминают `SQLAlchemy`.

Я выбрал реализацию на `Node.js + Express + Sequelize`, потому что это прямое требование основного ТЗ. При этом базу сделал на `PostgreSQL`, чтобы закрыть дополнительные требования по СУБД. `SQLAlchemy` не использовался осознанно, потому что он относится к Python-стеку и противоречит основному ТЗ на Node.js.

## Структура проекта

- `src/index.js` — точка входа, Telegram-бот и HTTP-сервер.
- `src/config` — загрузка `.env` и инициализация Sequelize.
- `src/models` — модели пользователей, FAQ, заявок и сообщений заявок.
- `src/services` — работа с пользователями, FAQ, заявками и сессиями.
- `src/scripts/seedFaq.js` — отдельное заполнение FAQ.
- `src/data/faq.seed.json` — стартовые FAQ-данные.
- `Dockerfile` — контейнеризация приложения.
- `.env.example` — пример переменных окружения.
- `requirements.txt` — формальный список зависимостей по дополнительному требованию.

Как реализовано

1. Сначала были разобраны оба документа с требованиями и выделены обязательные сценарии.
2. После этого была собрана модель данных:
   - `users` — пользователи Telegram и их роль;
   - `users.marketingOptIn/newsOptIn` — настройки подписки на рассылки;
   - `faq_sections` — разделы FAQ;
   - `faq_items` — вопросы и ответы;
   - `tickets` — заявки пользователей;
   - `ticket_messages` — все сообщения внутри заявки.
3. Дальше поднят каркас приложения:
   - `Express` для служебного HTTP-сервера;
   - `node-telegram-bot-api` для long polling;
   - `Sequelize` для работы с PostgreSQL.
4. Для FAQ сделаны два режима:
   - поиск по ключевому слову;
   - просмотр по разделам.
5. Для поиска по ключевому слову подключен `Fuse.js`, чтобы поиск был нечувствителен к регистру и переносил мелкие опечатки.
6. Для обратной связи сделана логика “открытой заявки”:
   - если открытая заявка уже есть, новые сообщения идут туда же;
   - если открытой заявки нет, создается новая;
   - первая новая заявка за день подтверждается отдельным сообщением.
7. Для администраторов сделан закрытый раздел:
   - бот показывает только открытые заявки;
   - админ выбирает заявку по `ID`;
   - следующий текст или файл бот отправляет именно пользователю этой заявки.
8. Для выполнения требования про отмену добавлен промежуточный этап:
   - при нажатии на кнопки бот сначала показывает “Выполняю запрос...” и кнопку `Отмена`;
   - если пользователь успевает нажать отмену, действие не выполняется.
9. По новому требованию добавлен раздел `Возможности` с быстрыми карточками о платформе, компетенциях, практиках, контактах, спринтах и октокоинах.
10. Для управления задачами и сроками добавлен HTTP API, через который внешняя платформа может отправлять Telegram-уведомления пользователям.
11. Для удобного деплоя добавлен `Dockerfile`.
12. После реализации были установлены зависимости, проверен синтаксис всех `.js`-файлов и подготовлен Docker-сценарий запуска.

## Переменные окружения

Проект использует `.env`. (можно -e в docker run)

Поля:

- `BOT_TOKEN` — токен Telegram-бота от `@BotFather`.
- `PORT` — порт для `Express`.
- `DB_HOST` — хост PostgreSQL.
- `DB_PORT` — порт PostgreSQL.
- `DB_NAME` — имя базы.
- `DB_USER` — пользователь базы.
- `DB_PASSWORD` — пароль базы.
- `DB_SSL` — использовать SSL для подключения к БД (`true/false`).
- `ADMIN_TELEGRAM_IDS` — список Telegram ID администраторов через запятую.
- `FAQ_SEARCH_LIMIT` — сколько результатов максимум показывать при поиске.
- `ACTION_DELAY_MS` — задержка для “динамической кнопки отмены”.
- `INTERNAL_API_TOKEN` — секрет для внутренних API-эндпоинтов уведомлений. Если поле задано, его нужно передавать в заголовке `x-internal-api-token`.

Пример:

```env
BOT_TOKEN=123456:telegram-bot-token
PORT=3000
DB_HOST=localhost
DB_PORT=5432
DB_NAME=oktabot
DB_USER=postgres
DB_PASSWORD=postgres
DB_SSL=false
ADMIN_TELEGRAM_IDS=111111111,222222222
FAQ_SEARCH_LIMIT=5
ACTION_DELAY_MS=700
INTERNAL_API_TOKEN=super-secret-token
```

## Локальный запуск без Docker

1. Создать `.env` по примеру `.env.example`.
2. Убедиться, что PostgreSQL доступен.
3. Установить зависимости:

```bash
npm install
```

4. Запустить приложение:

```bash
npm start
```

По умолчанию при старте:

- приложение синхронизирует таблицы;
- если FAQ пустой, он заполняется стартовыми данными из `src/data/faq.seed.json`.

docker run -d \
  --name oktabot-postgres \
  --network oktabot-net \
  -e POSTGRES_DB=oktabot \
  -e POSTGRES_USER=postgres \

## Новые разделы в боте

- `Возможности` — справочный блок про платформу, компетенции, практики, контакты, спринты и октокоины.
- `Рассылки` — управление подпиской на маркетинговые и новостные уведомления.

## Важно про уведомления

Бот поддерживает уведомления о решении задач, отклонении задач, новых задачах, спринтах, неактивности и завершении компетенций. Но автоматически они приходят только тогда, когда внешняя платформа или серверная часть Octagon вызывает внутренний API бота. Сам по себе Telegram-бот не может узнать о смене статуса задачи без интеграции с платформой.

## API уведомлений

Все методы ниже вызываются через `POST`. Если задан `INTERNAL_API_TOKEN`, добавляйте заголовок:

```bash
-H "x-internal-api-token: super-secret-token"
```

### Уведомление о принятии или отклонении задачи

```bash
curl -X POST http://localhost:3000/api/notify/task-status \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "telegramId": 726773708,
    "taskTitle": "Практика по SQL",
    "status": "approved",
    "comment": "Решение принято"
  }'
```

### Напоминание о неактивности

```bash
curl -X POST http://localhost:3000/api/notify/inactivity \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "telegramId": 726773708,
    "daysInactive": 3,
    "hint": "Зайдите на платформу и проверьте новые задачи"
  }'
```

### Завершение компетенции

```bash
curl -X POST http://localhost:3000/api/notify/competency-complete \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "telegramId": 726773708,
    "competencyTitle": "Backend Node.js"
  }'
```

### Уведомление о спринте

```bash
curl -X POST http://localhost:3000/api/notify/sprint \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "telegramId": 726773708,
    "type": "start",
    "sprintTitle": "Спринт 5",
    "message": "Стартовал новый спринт"
  }'
```

### Новая задача

```bash
curl -X POST http://localhost:3000/api/notify/new-task \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "telegramId": 726773708,
    "taskTitle": "Сделать Telegram-бота",
    "deadline": "2026-07-01",
    "sprintTitle": "Спринт 5",
    "description": "Реализовать FAQ и рассылки"
  }'
```

### Массовая новостная или маркетинговая рассылка

```bash
curl -X POST http://localhost:3000/api/broadcast/news \
  -H "Content-Type: application/json" \
  -H "x-internal-api-token: super-secret-token" \
  -d '{
    "category": "news",
    "title": "Новости платформы",
    "text": "Добавлен новый спринт и обновлены практики"
  }'
```

Значения `category`:

- `news`
- `marketing`
- `all`
